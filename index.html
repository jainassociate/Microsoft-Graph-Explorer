📖 Understanding Delta APIs

The Delta API allows you to maintain synchronization with files in a drive. Rather than iterating over all items in a massive folder list:

  1. You call the /delta endpoint initially to capture all existing files.
  2. The API response includes a state token (@odata.deltaLink).
  3. Future queries targeting this token only return modified, added, or deleted changes since your last check.
  4. Perfect for background syncing and notification processing systems.

📘 Microsoft Graph API Setup Guide

            
Complete manual to register your Azure App, grant correct OneDrive scopes, and generate active access tokens.


            

              
              

                1️⃣ App Registration in Microsoft Entra ID
                
                  Go to Microsoft Entra admin center or Azure Portal.
                  Go to Identity > Applications > App registrations > New registration. 
                  Provide a name (e.g., OneDrive Explorer), choose supported account types (e.g. Multi-tenant + Personal accounts), select Single-page application (SPA) or Web redirect type, and click Register.
                
              


              
              

                2️⃣ Configuring API Permissions (OneDrive Scopes)
                
                  In your new App Registration, go to API permissions > Add a permission > Microsoft Graph.

                  Select Delegated permissions (or Application permissions for daemon service scripts) and add these crucial scopes:
                  
    
                    
  • Files.ReadWrite.All — Required for folder listings, file creation, deleting, copying, and renaming.
    • 
                    
  • User.Read — Required to view Storage quotas, dashboard statistics, and login accounts.
    • 
                    
  • offline_access — Required to obtain refresh tokens for persistent background synchronization.
    • 
                  

                  Note: Remember to click Grant admin consent for [Tenant] if requested by your organization rules.

                4️⃣ Alternate Methods: Graph Explorer / Credentials Flow
                
                  Quickest Method: Navigate to the Microsoft Graph Explorer, log in, open the Access Token tab, copy the active token, and paste it on the left panel.
                  


                  Daemon / Client Credentials Flow (No Interactive User Consent):
                  Request a token in terminal using client_id and client_secret:
                  
curl -X POST https://login.microsoftonline.com/{tenant_id}/oauth2/v2.0/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "client_id={client_id}" \
  -d "scope=https://graph.microsoft.com/.default" \
  -d "client_secret={client_secret}" \
  -d "grant_type=client_credentials"
